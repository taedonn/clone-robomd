빈줄

- 그룹을 감싸고 있는 객체를 구분하기 위하여 코드 그룹 간 1줄씩 빈 줄을 만드는 것은 허용한다.
- 빈 줄의 간격이 1줄을 초과하지 않으며 개발과 연동되는 그룹의 경우는 빈줄보다 주석을 작성한다.

주석

- 코드의 가독성을 높이고 전체 HTML 구조를 쉽게 파악하기 위하여 주석사용을 준수한다.
- 주석 기호와 주석 내용 사이에는 반드시 공백 한 칸이 있어야 한다.
- 동일한 태그의 중첩으로 닫기 태그의 구분이 힘들어지는 경우 다른 작업자가 구분하기 쉽도록
   닫기 태그 뒤에 주석으로 해당하는 id/class를 표기한다.

마크업 유의사항

- 지키지 않을 경우 화면이 깨져보이거나 스크립트가 정상적으로 동작하지 못하는 등의 치명적인 오류가 발생할 수 있다.

- <head>태그의 안쪽과 <body>태그의 안쪽 이외에 내용을 삽입하지 않는다.

- <p>태그 안에는 p, div, ul 등 다른 블럭요소가 들어갈 수 없다. (display:inline 속성을 적용해도 소용없다.)

- <p>태그는 문단을 구분하는 용도로만 사용하고 영역을 구분하는 용도로 사용하지 말 것.

- <a>태그 안에는 <a>태그가 들어갈 수 없다.

- <form>태그 안에는 <form>태그가 들어갈 수 없다.

- <label>태그 안에는 <label>태그가 들어갈 수 없다.

- <pre>태그 안에는 <img>, <object>, <big>, <small>, <sub>, <sup>태그가 들어갈 수 없다.

- <button>태그 안에는 <input>, <select>, <textarea>, <label>, <button>, <form>, <fieldset>, <iframe>, <isindex>태그가 들어갈 수 없다.

- 하나의 ID는 페이지에서 한번만 사용한다. (스타일은 적용되나 책갈피 기능이나 스크립트를 정상적으로 사용할 수 없다.)

아이디/클래스 네이밍 규칙

- ID와 CLASS 구분

ID

1. 한 문서에 한 번만 사용할 수 있다.
2. 링크에 대한 앵커(목표위치)로 사용할 수 있다.
3. 클래스에 지정한 CSS값보다 ID에 지정한 CSS값이 우선적용된다.

CLASS

1. 한 문서에 여러번 사용할 수 있다.
2. 하나의 객체에 여러개를 사용할 수 있다.

ID/CLASS 권장 기준

1. 한 문서에서 반드시 한 번만 사용 확신 + 앵커로서 가치가 있다 = ID
2. 한 문서에서 두 번 이상 사용 가능성 + 앵커로서 가치가 없다 = CLASS
3. 한 문서에서 두 번 이상 사용 가능성 + 앵커로서 가치가 있다 = ID/CLASS 함께 사용

서비스 구분

- 서비스 별로 스타일을 구분할 수 있게 해당 서비스를 특정할 수 있는 네이밍을 body태그의 자식 요소에 지정한다.

네이밍 규칙: 언더바

- 스타일 적용을 위한 아이디, 클래스의 경우 단어와 단어 사이를 언더바(_)로 구분한다.

네이밍 규칙: 카멜케이스

- 자바스크립트 적용을 위한 아이디, 라벨 연결을 위한 입력폼의 아이디의 경우 카멜케이스의 규칙을 따른다.
※ 카멜케이스 : 영문소문자를 사용하되 두번째 단어부터 첫번째 문자를 대문자로 처리하는 방법.

네이밍 규칙: 용도에 맞는 단어 사용

- 아이디/클래스 이름은 용도를 나타내는 단어를 사용하되, 위치/크기/색상 등의 스타일을 특정짓는 단어를 지양한다.

네이밍 규칙: 첫 단어

- 아이디/클래스 이름은 대문자, 숫자, 특수문자로 시작할 수 없다.

네이밍 규칙: 숫자로 순서 구분

- 유사한 용도인 경우 가장 뒤에 숫자를 붙여 사용할 수 있으며 첫번째 이름은 숫자를 생략할 수 있다.

네이밍 규칙: 네이밍 순서

- 네이밍의 조합은 형태 => 의미 => 순서 => 상태를 기본 순서로 사용한다.

스타일 선언

- 스타일가이드에 나와있는 공통 디자인이 아닌 경우, 특정 페이지를 구분할 수 있는 부모요소없이 단독으로 스타일을 선언하지 않는다.

CSS 코드 작성 규칙 : 대소문자 구분

- 자바스크립트 적용 등을 위한 특수한 경우를 제외하고, 모든 속성은 영문 소문자로만 작성

CSS 코드 작성 규칙 : 세미콜론(;)

- 마지막에 선언된 속성에도 세미콜론(;)을 사용한다.

CSS 코드 작성 규칙 : 공백의 사용

- 선택자 간, 중괄호간 한칸의 공백을 사용한다, 중괄호 안쪽 좌우에는 공백은 사용하지 않는다.

JS 코드 작성 규칙 : 카멜케이스

- JS 코드 작성 시 함수, 변수의 네이밍은 카멜케이스를 기본으로 한다.

JS 코드 작성 규칙 : 공백의 사용

- 함수 사용 시, 중괄호간 한칸의 공백을 사용한다. 중괄호 이외에 대괄호, 소괄호에는 공백을 사용하지 않는다.
- 콤마(,) 뒤에는 반드시 공백을 사용한다.
- 콜론(:)을 사용하는 경우 반드시 앞 뒤에 공백을 사용한다.
- 연산자(>,<,=)의 앞 뒤에는 반드시 공백을 사용한다.

JS 코드 작성 규칙 : 세미콜론(;)

- 변수 선언, 함수 선언, 함수 실행, 소괄호 사용 시 반드시 세미콜론을 사용하여 끝맺는다.

JS 코드 작성 규칙 : 주석

- 주석은 되도록 한줄로 사용한다.
- 주석 다음줄은 공백을 사용하지 않는다.
- 코드에 설명이 많이 담긴(explanatory) 주석 사용을 지양한다.
- 주석을 많이 사용하기보다 코드 자체를 이해하기 쉽게 작성하는 것을 지향한다.
